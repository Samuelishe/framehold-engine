# SECURITY_NOTES

## Минимальные правила безопасности

- Не хранить секреты в Git.
- Не коммитить `.env`-файлы.
- Не коммитить дампы БД.
- Не коммитить приватные ключи.
- Не коммитить приватные медиафайлы.
- Не реализовывать собственное password hashing.
- Не реализовывать собственную auth/session систему поверх стандартной без отдельного решения.
- Использовать механизмы Django/Wagtail для пользователей, сессий, прав и административной инфраструктуры.
- Contributor не должен иметь возможность редактировать чужой контент.
- Публичные пользователи не должны видеть `draft` и `hidden` контент.
- Публичные загрузки файлов не допускаются.
- Development и production секреты должны быть разделены.

## Практический смысл

Даже на ранних этапах проект должен быть организован так, чтобы случайно не утекали реальные данные, не ослаблялись проверки доступа и не появлялась самодельная auth-логика без необходимости.
